  # Localize - Making FormatJS & i18n EASY.
  <p>{`This tool handles storage and provides adapters to simplify implementation of i18n / FormatJS in your application`}</p>
  
  ### Do I need this library? 
    Since Localize simply implements FormatJS/i18n, all of the functionalities we rely on come out of the box from those libraries. 
    Localize just makes it dead simple to implement by handling storage and some of the common implementation patterns.
    
  ## Client-side rendering
  <p>{`For client-side rendering, the built-in singleton (as shown in this storybook) should solve 95% of your usecases`}</p>

## Server-side rendering

    <p>{`For server-side rendering, you'll need to create an instance so that you don't run into server-side cache problems`}</p>

    <pre>
        <code>
            {`
            // utilities/localize/instance.ts

            import { cache } from "react";
            import { createLocalize } from "localize";

            export const localize = cache(() => createLocalize());
        `}
        </code>
    </pre>

## Fetching, Validating & Storage

  <p>
    {`Whether you're using the singleton or the instance, you then need to fetch your localized content from your api, set your locale and save the messages into storage.`}
  </p>
  <pre>
    <code>
    {`
        // async.ts

        import { localize } from "@/utilities/localize/instance";

        export const getLocalizedContent = async (locale: string) => {
            const response = await fetch(url); // url is your api endpoint

            // (optional) validate response using zod
            const ContentAPIResponse = z.object({
                example: z.string().default("Fallback value")
            });

            const validatedResponse = ContentAPIResponse.safeParse(response);

            if (!validatedResponse.success) {

                // handle your error however works best for your application
                log.error("Error parsing content", { ...validatedResponse.error });

                return;
            } else {

                // check existing locale in Localize storage
                const l = localize().getLocale();

                // if new locale, this clears out existing messages
                if (l !== locale) {
                    localize().setLocale(locale);
                }
                // add new localization data to storage
                localize().appendMessages(validatedResponse.data);
            }
        }
    `}

    </code>

  </pre>

## Implementation

  <p>{`From there, you simply import your localize instance across your application`}</p>
  <pre>
    <code>
    {`
        // Component.tsx
        import { localize } from "@/utils/localize/instance";

        const hello-msg = localize().formatMessage("example/hello");
    `}
    </code>

  </pre>
